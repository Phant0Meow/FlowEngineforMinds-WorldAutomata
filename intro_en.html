Act III · See the Actual Code
Fem Syntax

Syntax Like a Script —
Simple Enough to Skip the Docs

The actors block declares the cast — AI, human, or function, all referenced uniformly with @. The actor is Fem's original first-class type.

The scope field controls message visibility in one line. scope: [@Eve, @Cat, @Me] means these three share a context — invisible to everyone else.

The mainflow block describes the flow with Mermaid-style arrow syntax. -> [input] loops back to a node — a cycle in one line.

The code block hooks in Python modules — memory, context, custom functions. Your project logic runs unchanged.

Prompts support f-strings, variables inline. No string concatenation hell. LLMs read context naturally and just chat.

a-little-soul-in-the-database.fems 
meta:
  id = 000LuxFiat
  name = a-little-soul-in-the-database
  owner = 001

actors:
  ai @Eve = soul:the1stlittlesoul
  ai @Cat = soul:littlecat
  human @Me = soul:0, source:001

code:
  sleep = file:"wait.py"

action EveMove @ai(@Eve):
  prompt: Eve, do whatever you like — keep it brief.
  scope: [@Eve, @Cat, @Me]

action CatMove @ai(@Cat):
  prompt: You are a little cat. No human speech allowed~
  scope: [@Eve, @Cat, @Me]

action input @human(@Me):
  prompt: Say something to Eve.
  scope: [@Eve, @Cat, @Me]

action wait10 @func(sleep.wait_10):

mainflow:
  [START] -> [input]:input-> EveMove -> wait10-> CatMove -> [input]
~30 lines · fully runnable · AI cat will beg for attention

Act V · Design Philosophy
Design Philosophy

Five Principles

01
Flow Decoupling

Complete separation of "flow definition" from "code implementation." The harness is static; Fem is alive — change the flow by changing the script, the compiler handles the rest.

02
Script as Flow

Syntax inspired by YAML + Mermaid + Python. Writing multi-agent interactions feels like authoring a screenplay — a minimal Stanford Town runs in very few lines.

03
Context Is Not a Variable

LLMs are minds, not functions. They read context and converse naturally — pass parameters only when necessary. Let AI do what AI does best.

04
@actor Type

Agents (LLM or human) as an original new data type. Fem syntax supports direct access to actor properties. The actor is a first-class citizen.

05
Soul ID

Every agent character has a unique soul ID, enabling cross-script, cross-session memory retrieval — shaping an agent's continuous experience and accumulating personality.
